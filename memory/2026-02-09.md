## 🔧 Debugging & System Fixes

### Server Exit Issue - RESOLVED ✅

**Problem:** Development servers would start successfully but then exit immediately when run in background with `&`

**Root Cause:** Using `&` alone to background processes caused them to be killed when shell session ended

**Solution:** Use `nohup` command to keep processes running in background:
```bash
cd /home/nosyt/.openclaw/workspace && nohup ./node_modules/.bin/next dev -p 3000 > /tmp/oma-ai-bg.log 2>&1 &
cd /home/nosyt/.openclaw/workspace/spendthrone && nohup ./node_modules/.bin/next dev -p 3001 > /tmp/spendthrone-bg.log 2>&1 &
cd /home/nosyt/.openclaw/workspace/lethometry && nohup ./node_modules/.bin/next dev -p 3002 > /tmp/lethometry-bg.log 2>&1 &
```

**Result:** All 3 sites now running stably

### Lessons Learned

1. **Use nohup for background processes** - Always use `nohup` when starting dev servers in background
2. **Check lock files** - Clear `.next/dev/lock` files when servers fail to start
3. **TypeScript needs explicit paths** - Ensure `@/` alias is configured in `tsconfig.json`
4. **Log output is critical** - Always redirect output to log files when backgrounding processes
