x402 Payment Protocol

OMA-AI uses the x402 protocol developed by Coinbase for instant, trustless micropayments over HTTP.

How It Works

  1. Client requests a resource from the server
  2. Server responds with 402 Payment Required and payment instructions in the PAYMENT-REQUIRED header
  3. Client creates a payment payload and sends it via the PAYMENT-SIGNATURE header
  4. Server verifies payment via the facilitator and returns the resource
# Request without payment
curl https://oma-ai.com/api/search?q=bitcoin
# HTTP 402 with PAYMENT-REQUIRED header

# Request with x402 payment
curl -H "PAYMENT-SIGNATURE: " \
     https://oma-ai.com/api/search?q=bitcoin
# HTTP 200 with results

Example: Client with x402

import { x402Fetch } from '@x402/fetch';
import { createEVMWallet } from '@x402/evm';

const wallet = createEVMWallet(process.env.PRIVATE_KEY);
const fetch = x402Fetch({ wallet });

// Automatically handles 402 responses and payments
const response = await fetch('https://oma-ai.com/api/search?q=bitcoin');
const data = await response.json();
